## Context

系统当前只在 `holdings` 中保存最新价、历史最高价和当前止损价，行情刷新会覆盖旧值，因此无法直接绘制真实价格趋势。持仓详情已具备响应式摘要和异步状态模型，但前端尚未引入图表依赖。股票实时价和开放式基金净值的更新频率与交易日语义不同，历史查询必须统一成按市场自然日的数据，同时保留数据来源和时区信息。

## Goals / Non-Goals

**Goals:**

- 为单个持仓提供可缓存、可增量更新的日级历史价格序列。
- 为固定价格、百分比和移动止损生成可解释的逐日止损线。
- 在桌面和手机详情页中以简洁图表展示价格、买入基准、止损边界和触发点。
- 外部行情失败时使用已有缓存，避免清空图表或阻塞持仓详情主体。
- 控制新增前端依赖与历史查询的数据量。

**Non-Goals:**

- 不提供分时、分钟级行情或实时推送。
- 不加入 K 线、成交量、均线、KDJ 等技术分析指标。
- 不将历史图表作为自动交易或回测结果。
- 不保证补齐外部数据源本身缺失或停牌日期的数据。

## Decisions

### 1. 使用独立的日行情缓存表

新增 `price_history`，至少保存资产代码、资产类型、交易日期、收盘价或单位净值、来源和抓取时间，并以 `(code, asset_type, trade_date)` 建立唯一约束。历史行情属于资产级数据，可供多个同代码持仓复用，不与持仓生命周期绑定。

选择本地缓存而不是每次透传 AkShare，是为了降低详情页延迟和外部服务波动；不直接记录每次轮询快照，因为分钟级快照会快速膨胀且不能为已有持仓提供过去行情。

### 2. 查询时按需补齐并增量更新

新增持仓历史接口，接受受限范围 `1m|3m|6m|1y`，默认 `3m`。服务先读取缓存，再根据请求范围和最后交易日期判断是否需要调用历史行情适配器；成功结果以 upsert 写入，外部请求失败但缓存非空时返回缓存及 `stale` 元数据，完全无数据时返回可恢复错误或空数据状态。

接口返回已经规整、升序且去重的日线数据，前端不直接理解 AkShare 字段。为避免慢请求拖垮页面，持仓主体与历史图表使用独立请求状态。

### 3. 后端生成止损序列

接口根据持仓买入日过滤价格序列，并返回每个交易日对应的 `price` 与 `stop_loss_price`：

- 固定止损：始终使用配置的固定价格。
- 百分比止损：始终按买入价和百分比计算。
- 移动止损：从买入价开始维护逐日运行最高价，按该最高价计算只升不降的阶梯止损线。

由后端复用止损引擎生成序列，可以避免前后端规则漂移。当前配置变更前的历史止损参数没有被版本化，因此图表按“当前止损配置应用到整个可见区间”展示，并在图表说明中明确这一语义；不伪造过去实际配置。

### 4. 使用按需引入的 Apache ECharts

新增独立 `HoldingPriceChart` 组件，按需注册折线、标记、提示框和数据缩放能力。价格使用普通折线，移动止损使用阶梯线，买入价使用低对比度虚线，止损线以下使用淡红风险区域；触发日使用带文字的标记点。

选择 ECharts 是因为阶梯线、markLine/markArea、触控提示和响应式支持成熟。相比自行绘制 SVG，它能减少交互和可访问性边界错误；相比完整引入，按需模块可以继续接受现有包体预算约束。

图表外必须提供文字摘要（时间范围、最高/最低、最新价、当前止损价），保证风险信息不只依赖颜色或画布。

### 5. 保持详情页视觉层级克制

图表位于风险摘要之后、止损设置之前，默认高度桌面约 320px、手机约 240px。范围切换使用紧凑分段控件；首次加载显示骨架，刷新期间保留已有图形，缓存过期或刷新失败显示轻量提示。价格和止损使用现有资产精度格式化规则。

## Risks / Trade-offs

- [AkShare 历史接口字段或稳定性变化] → 为股票和基金建立独立适配器与固定契约测试，错误时保留缓存并暴露数据状态。
- [首次查询延迟较长] → 图表独立异步加载、限制最大一年范围，并在成功后复用缓存。
- [当前止损配置不能还原历史真实配置] → 明确标注为“按当前规则模拟的止损线”，未来若需要审计再引入配置版本表。
- [基金净值不是盘中实时价] → 图表标注来源和最后净值日期，不将抓取时间冒充净值日期。
- [图表库增加包体] → 按需导入并继续执行生产包体预算检查。
- [同代码资产缓存冲突] → 唯一键包含资产类型，历史适配器输出统一的 Decimal 精度和市场日期。

## Migration Plan

1. 增加 `price_history` 表迁移和唯一索引；迁移只新增结构，不修改现有记录。
2. 发布后按用户首次访问范围惰性填充历史数据，无需一次性全库回填。
3. 后端接口稳定后启用详情页图表；旧客户端不调用新接口，保持兼容。
4. 回滚前端时历史表可安全保留；回滚后端时不删除表，避免破坏已缓存数据。

## Open Questions

- AkShare 对不同基金分类的历史净值接口覆盖率需要在实现阶段用现有目标基金做一次适配验证。
- 图表标题最终使用“止损警告线”还是更准确的“止损线（按当前规则）”，建议采用后者并在触发区域使用“风险区”。
