## Context

这是一个单用户本地 FastAPI/Vue 应用，数据保存在 SQLite。当前会在导入时建表、在可热重载 Web 进程内运行 APScheduler、每次启动安装依赖、强制终止占用端口的任何进程、监听全部网卡，且后端仅有公式测试。前两个变更会扩大数据模型与公开契约，进一步提高迁移、恢复和回归验证要求。

## Goals / Non-Goals

**Goals:**

- 所有数据库变更都显式、可备份、可验证、可回滚。
- 支持的本地部署中只有一个调度器所有者，并能优雅启停。
- setup、开发、正常启动、停止、备份、恢复、验证职责分离且安全。
- 提供不泄露持仓信息的结构化诊断。
- 建立分层自动化门禁和前端包体预算。

**Non-Goals:**

- 不提供高可用、分布式调度、云部署或远程多用户访问。
- 不替换 SQLite 或 akshare。
- 不重复实现前两个变更中的业务逻辑。
- 不承诺本地数据库升级零停机。

## Decisions

1. **使用版本化迁移。** 引入 Alembic 风格迁移，正常启动不再执行 `create_all`。每个 revision 支持状态检查，并在可行时提供 downgrade。
2. **使用 SQLite 在线备份 API。** 备份生成新时间戳文件及 manifest，记录 checksum、schema revision 和时间；恢复前在临时副本上校验完整性，服务运行时不得直接替换数据库。
3. **单一调度器所有者。** 正常启动使用单 worker、无 reload，并在 FastAPI lifespan 中启停调度器；开发 reload 默认关闭调度器，只有显式选择才开启。
4. **安全进程脚本。** 默认绑定 `127.0.0.1`；端口占用时报告 PID 与进程名并退出，绝不自动杀进程。停止时校验 PID 的命令与工作目录确属本项目。
5. **setup 与 runtime 分离。** setup 安装锁定的运行依赖及独立测试依赖；start 只验证环境，缺失时输出可执行的 setup 命令。
6. **隐私友好的结构化日志。** 日志包含时间、级别、组件、关联/周期 ID、耗时、计数和错误类别，默认不记录价格、数量、成本或完整响应体。liveness 检查进程；readiness 检查 schema、数据库和调度器，不依赖外部行情源。
7. **默认离线的测试金字塔。** API 使用临时 SQLite，行情适配器使用 fixture，调度器注入时钟，前端使用 fake timers 和 mock API；真实行情测试仅显式运行。
8. **包体预算而非隐藏警告。** Element Plus 与图标按需导入，路由继续懒加载；入口或路由产物超出预算时构建失败，调整预算必须有说明。

## Risks / Trade-offs

- **迁移工具增加维护成本** → 保持 revision 小而清晰，并对代表性数据库测试升级和降级。
- **开发 reload 默认不再监控** → 启动时醒目提示，并提供隔离的显式开关。
- **PID 可能被系统复用** → 停止前校验命令行和工作区标识，不匹配则只报告。
- **行情 fixture 可能落后上游** → 适配器报错必须可诊断，并保留独立真实行情冒烟测试。
- **包体预算可能过于敏感** → 只约束稳定入口/路由产物，记录基线，变更预算需要理由。

## Migration Plan

1. 引入类型化配置、结构化日志、应用工厂、测试注入和健康检查。
2. 加入迁移工具并建立当前 schema 基线，在数据库副本上验证升级/降级。
3. 加入备份恢复命令，非基线升级前强制先生成有效备份。
4. 拆分 setup/dev/start/stop，启用 loopback 默认值和调度器所有权检查，再移除导入时建表。
5. 逐层补齐测试与包体门禁，最终收敛为一个验证命令。
6. 回滚时同步回滚脚本和应用；schema 不兼容则恢复升级前备份。
