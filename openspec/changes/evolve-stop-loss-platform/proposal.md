## Why

当前系统已经具备本地止损监控 MVP 的完整骨架，但行情占位值与真实行情缺少明确区分、监控周期仍存在外部日历耦合和并发一致性风险，持仓触发后缺少重新布防等处置路径，页面也尚未提供仓位权重、绝对风险、监控健康度和完整历史等决策信息。现在需要在保持“本地运行、不连接券商、不自动下单”边界的前提下，把系统从记录与提醒工具演进为行情可信、事件可追溯、风险优先且可继续扩展的个人止损管理平台。

## What Changes

- 建立明确的行情状态模型，区分未取价、实时、延迟、收盘价、基金净值、过期和失败，所有止损判断只接受满足资产策略的新鲜行情。
- 解耦交易日历与行情提供方，增加缓存、超时、退避、降级和确定性测试适配器，保证必选质量门禁完全离线。
- 原子化持仓触发与告警创建流程，支持并发刷新，并保证响应只报告已提交的状态变化。
- 扩展触发后的处置流程，支持确认风险、重新布防和确认平仓，并为人工覆盖保留原因及审计记录。
- 将标的、交易批次、汇总仓位、止损规则、行情快照和领域事件划分为清晰边界；首轮保留现有持仓 API 兼容层。
- 增加部分平仓、平仓时间、费用税费、可配置数量精度、绝对盈亏和已实现盈亏计算。
- 保存价格、止损线、规则变化和生命周期事件历史，为详情趋势图、复盘和未来回测提供数据基础。
- 增加持仓/告警筛选分页、行情与调度诊断、仓位权重、预计止损损失、移动端紧凑模式和关闭持仓专属详情。
- 增加可选的系统通知、Webhook 等通知通道，站内告警继续作为唯一事实来源，外部投递失败不得影响止损状态提交。
- 增加 CSV 导入、导出和可验证的数据备份入口，批量导入先预览并逐行报告错误。
- 强化迁移、恢复、浏览器 E2E、并发和数据提供方契约测试，修复当前冒烟测试仍隐式依赖网络的问题。
- **BREAKING**：内部数据模型将由单一 `Holding` 逐步迁移为标的、交易批次、仓位和事件模型；数据库迁移必须自动保留现有数据。公开 HTTP API 在一个兼容周期内保持现有字段和路由可用。

## Capabilities

### New Capabilities

- `monitoring-diagnostics`: 监控周期、行情状态、提供方健康、最近成功时间和可操作诊断。
- `portfolio-accounting`: 交易批次、部分平仓、费用税费、仓位汇总以及已实现/未实现盈亏口径。
- `position-history`: 行情、止损线、规则变更和持仓生命周期事件的持久化历史与查询。
- `notification-delivery`: 站内告警之外的系统通知和 Webhook 投递、重试、幂等及隐私控制。
- `data-portability`: CSV 预览导入、导出和面向用户的数据备份能力。

### Modified Capabilities

- `price-fetching`: 增加可注入交易日历、明确行情状态、缓存降级、提供方超时以及完全离线的 fixture 行为。
- `stop-loss-engine`: 增加原子触发提交、触发后确认/重新布防语义及完整审计要求。
- `holdings-crud`: 增加行情状态、交易批次、部分平仓、关闭时间、费用和关闭持仓正确展示。
- `alert-system`: 增加完整触发快照、筛选分页、处置关联和异步外部投递语义。
- `dashboard`: 增加监控健康、仓位权重、绝对风险、预计止损损失和可信行情口径。
- `frontend-shell`: 增加统一筛选、状态图例、密度规则、系统通知权限和进一步的可访问性要求。
- `runtime-settings`: 增加行情、历史保留、通知通道等设置的分组、校验和运行时应用规则。
- `local-operations`: 增加真正有序且可验证的迁移、恢复兼容校验、缓存清理和诊断导出。
- `quality-gates`: 增加完全离线冒烟、真实组件测试、浏览器 E2E、并发触发和迁移数据保真门禁。

## Impact

- 后端：`models.py`、`schemas.py`、`migrations.py`、行情/监控/止损服务、全部业务路由、调度器和运维脚本。
- 前端：应用外壳、仪表盘、持仓列表与详情、告警、设置、Pinia 状态、格式化与轮询工具。
- 数据库：新增标的、交易批次、平仓分配、行情历史、止损规则、领域事件、监控周期和通知投递表；需要从 schema v2 执行可回滚迁移。
- API：优先新增字段和新资源路由；旧持仓与仪表盘字段至少保留一个兼容周期并记录弃用计划。
- 依赖：可考虑 Alembic 管理迁移、浏览器 E2E 工具和图表组件；任何新增依赖需证明必要性并纳入包体预算。
- 安全与隐私：继续默认只监听 loopback；外部通知默认关闭，敏感价格、数量和持仓名称只有在用户明确配置后才允许发送。
