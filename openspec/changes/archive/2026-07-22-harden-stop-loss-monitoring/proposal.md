## Why

当前监控链路可能误判交易日、在非交易时段轮询、重复下载全市场行情、错误关联同代码持仓，并在行情失败后用旧价格判断止损。该链路直接产生投资告警，必须先做到结果确定、数据时效可判定、全过程可追踪，才能用于真实持仓。

## What Changes

- 统一交易日数据格式，并按 Asia/Shanghai 时区严格限制 A 股交易时段。
- 将逐持仓拉取改为按行情数据集批量拉取，再把结果分发给所有匹配持仓。
- 记录行情来源、行情时间、抓取时间和新鲜度；失败或过期行情不得触发止损。
- 将状态流转改为 `holding → triggered → closed`，明确区分“触发信号”和“确认成交”。
- 金额和价格改用十进制定点精度，止损比较遵循统一舍入规则。
- 告警保存持仓快照，并通过数据库约束保证同一触发生命周期只产生一次告警。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `price-fetching`：补充交易日/交易时段、真实批量抓取、行情新鲜度和失败处理要求。
- `stop-loss-engine`：补充十进制计算、触发与成交分离、幂等触发要求。
- `alert-system`：补充告警快照、历史保留和重复告警防护要求。

## Impact

- 影响持仓与告警数据模型、数据库迁移、行情服务、调度器、止损服务、手动刷新、仪表盘和告警接口。
- 现有 SQLite 数据需要迁移：`stopped_out` 将回填为 `closed`，金额字段转换为定点数。
- API 使用方需要识别 `triggered` 与 `closed`；本变更不引入自动下单。
